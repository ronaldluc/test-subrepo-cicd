11 of 20 user stories produced a clean pass. The 5 that came back blocked all hit the same wall: the fine-grained PAT in use grants contents:write but not Administration, Pull requests, or Actions — so the test harness can’t create a ruleset, open a PR, or dispatch a workflow. Everything the API does let us do worked.

What worked — the headline findings

Skipped-as-success path-filtered CI

The plan’s “ci-gate’s if: always() reports green when no prod paths changed” works only if the workflow always starts. Top-level paths: filters quietly defeat required-check enforcement. We refactored each workflow to start always and put the path test inside a dorny/paths-filter job. On the spike-only push to main:

prod-ci run:
  changes: success
  lint:    skipped       ← no prod files
  ci-gate: success       ← skip-as-success ✓
ops-ci  run:
  changes: success
  lint:    skipped
  ops-gate: success
spikes-smoke run:
  smoke:   success       ← real run

LFS round-trip is clean

A 5 MB clip.mp4 committed locally became a 132-byte pointer in git but a 5 242 880-byte binary in the working tree. CI shows the exact same split:

with-lfs:  5,242,880 bytes  "data"
without:        132 bytes  "ASCII text"
            version https://git-lfs.github.com/spec/v1

Fresh clone into /tmp/cicd-fresh retrieved the full blob automatically — no pointer leaked through.

Pre-commit fast/heavy split

trailing-whitespace fires on a spikes/ file (fast stage runs everywhere). ruff with files: ^prod/ reports “(no files to check)” on an ops/ file and a hard 3-error fail on the exact same file relocated to prod/. Heavy stage is properly scoped.

Large-file rejection

A 2 MB random blob.bin staged in prod/ is rejected at commit time:

check for added large files .... Failed
prod/mock-app/blob.bin (2048 KB) exceeds 1000 KB.

Dependabot wakes up fast

The migration plan budgeted 24 h for Dependabot to register. In practice, PR #1 opened within minutes of pushing .github/dependabot.yml — build(deps): Bump lodash from 4.17.0 to 4.18.1 in /prod/mock-app. spikes/ was not listed in the config and got no PR, as designed.

AGENTS.md symlinks survive git

Every CLAUDE.md has a sibling AGENTS.md → CLAUDE.md stored as a real git symlink (mode 120000). Delete one and the local guard fails with exit 1; restore it and it passes. The hook works.

Blockers & what to feed back to the migration plan

  1. Fine-grained PAT scope. The token in use here can push code but cannot create rulesets, open PRs, or dispatch workflows. Either regenerate the PAT with Administration: write + Pull requests: write + Actions: write, or do those steps manually in the UI.
  2. Top-level paths: filters break required checks. If prod-ci is gated by paths: [prod/**] at the workflow level and only spikes/** changes, the workflow never starts and the required check appears as expected/missing indefinitely. Move the filter into an inner changes job so the gate job always reports. The plan’s “skipped-as-success” sentence is correct in intent but glosses over this.
  3. Dependabot wakes up immediately. The README budgets 24 h. Real-world latency for this repo was minutes. Tighten that timeline.
