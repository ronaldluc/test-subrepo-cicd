Monorepo migration — step-by-step plan

End-to-end procedure for taking conjurehq/conjure and turning it into a per-path-protected monorepo per the design in results.html and first-principles.html. Every step lists the exact command, the exact GitHub URL, and what to verify before moving on.

Variable placeholders used below: <ORG> = conjurehq, <REPO> = monorepo (suggested; you can keep conjure if you want to restructure in place), <USER> = ronaldluc.

Phases

  1. Pre-flight decisions & account inventory
  2. Phase 0 — Bot accounts and PATs
  3. Phase 1 — Create the empty monorepo on GitHub
  4. Phase 2 — Import conjure into prod/conjure_tech/ with full history
  5. Phase 3 — Scaffold directory tree + CLAUDE.md hierarchy
  6. Phase 4 — LFS, .gitattributes, R2 setup
  7. Phase 5 — Pre-commit (fast + heavy)
  8. Phase 6 — CI workflows + skip-as-success gate
  9. Phase 7 — CODEOWNERS + branch ruleset
  10. Phase 8 — Dependabot
  11. Phase 9 — ai-cannot-approve workflow

  16. Phase 14 — End-to-end validation against US-1..US-30

Time budget: 2.5–3 focused days of work for one engineer. Phases 0, 4, 10 each have asynchronous waits (token issuance, R2 propagation, CodeRabbit install).

Phase 0 — Bot accounts and PATs

~45 min · do this first; token issuance is bursty

0.1 — Create the Claude review bot user account

  1. Sign out of your personal GitHub account in a private browser window.
  2. Go to https://github.com/join and create a new user. Suggested handle: conjure-reviewer. Use a dedicated email (e.g. reviewer-bot@).
  3. Verify the email. Enable 2FA (required for org membership).
  4. Invite the bot to the org: https://github.com/orgs/conjurehq/people → Invite member. Set role: Member (not Owner).
  5. From the bot account, accept the invite. Verify: gh api users/conjure-reviewer --jq .type returns "User" (not "Bot"). This is what lets its comments pass the ai-cannot-approve guard.

Phase 2 — Import conjure into prod/conjure_tech/

~30 min · the only step where you can lose history if you fat-finger

2.2 — Move every file under prod/conjure_tech/ with full history

git filter-repo rewrites every commit so paths are prefixed. Order matters: run only on the working copy, never on a checkout that other things share.

4.1 — LFS patterns

The .gitattributes copied in Phase 3.3 already routes *.mp4 *.mkv *.glb *.safetensors to LFS. Add anything else you currently version that’s > 2 MB and isn’t text. Suggested additions for conjure:

4.2 — Create the R2 ops bucket

UI Cloudflare dashboard → R2 → Create bucket. Name: conjure-ops. Same account/region as the existing conjure-artifacts bucket. Direct URL: https://dash.cloudflare.com → R2 Object Storage → Create bucket.

4.3 — Scoped API token for CI (no Delete)

UI R2 → Manage API Tokens → Create API Token:

  • Name: monorepo-ops-rw-no-delete
  • Permissions: Object Read & Write (NOT Admin Read & Write)
  • Buckets: conjure-ops only
  • TTL: 1 year

Save the access key, secret key, and S3 endpoint URL. Add them to repo secrets (Phase 1.5) as R2_OPS_ACCESS_KEY, R2_OPS_SECRET_KEY, R2_OPS_ENDPOINT.

Critical check: from a shell with those credentials, run aws s3 rm s3://conjure-ops/test-object --endpoint-url=<R2_ENDPOINT>. It should fail with 403 Forbidden. If it succeeds, the token has Delete permission and you need to re-mint it with the right scope.

11.1 — docs-sanity.yml

cat > .github/workflows/docs-sanity.yml <<'EOF'
name: docs-sanity
on:
  pull_request:
    paths: ['docs/**']
permissions:
  pull-requests: write
  contents: read
jobs:
  sanity:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0 }
      - name: Compute deletion stats
        id: stats
        run: |
          git diff --numstat origin/${{ github.base_ref }}...HEAD -- docs/ \
            | awk '{added+=$1; deleted+=$2} END {print "deleted="deleted; print "files="NR}' >> $GITHUB_OUTPUT
      - name: Run Haiku sanity check
        if: steps.stats.outputs.deleted > 200 || steps.stats.outputs.files > 5
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          # call Claude Haiku via the API with the diff + PR title
          # output YES/NO + reason; exit 1 on NO
          ./scripts/docs-sanity.sh
EOF

12.2 — Move with history

Use git filter-repo on a temporary branch to relocate files while preserving blame. Two approaches:

Phase 13 — Per-product make test + root dispatcher

~2 hours

The root Makefile is a dispatcher; each product owns its real targets.

cat > Makefile <<'EOF'
# Root dispatcher. Args:
#   PATHS=   (CI passes this)
#   PRODUCT=                          (manual override)

PATHS ?=
PRODUCT ?=

.PHONY: test lint typecheck

test:
ifdef PRODUCT
	$(MAKE) -C prod/$(PRODUCT) test
else
	@for p in $$(./scripts/products-from-paths.sh $(PATHS)); do \
	  $(MAKE) -C prod/$$p test; \
	done
endif

lint:
ifdef PRODUCT
	$(MAKE) -C prod/$(PRODUCT) lint
else
	@for p in $$(./scripts/products-from-paths.sh $(PATHS)); do \
	  $(MAKE) -C prod/$$p lint; \
	done
endif

typecheck: ## same shape as test/lint
EOF

Each prod/<product>/Makefile defines real targets. For conjure_tech, port the existing Makefile targets from the old conjure repo.

scripts/products-from-paths.sh reads a list of paths, prints the unique prod/<X> roots that contain them. Trivial bash.

Wire CI: in .github/workflows/prod-ci.yml, the lint job runs make test PATHS="${{ steps.changes.outputs.touched_files }}".

Failure modes and rollback

If …Then …
filter-repo produces a broken historythrow away conjure-import/ and start Phase 2 again. The new repo on GitHub stays empty until you push. Idempotent.
Ruleset blocks every PR (required checks misnamed)delete the ruleset (gh api -X DELETE /repos/…/rulesets/<id>); fix the contexts; re-create. You retain admin so nothing locks you out.
R2 token has Delete permission by mistakerevoke at the Cloudflare dashboard; re-mint with correct scope; update repo secrets.
CodeRabbit floods PRs with noisetune .coderabbit.yaml path filters; if unsalvageable, uninstall the App and stick with Claude-deep-review only.
claude-deep-review costs spikelower --max-cost-usd in the workflow; set a daily org-wide Anthropic spending cap; disable on non-prod PRs.
Production deploy regresses after Phase 15.2roll back the deploy via Netlify/Wrangler/Railway; the new repo stays as-is, old conjure repo isn’t archived yet so it can resume serving.

Open questions that need human decisions before starting

  1. Do we want a second human reviewer on prod/ from day one, or accept solo-codeowner no-op for the first weeks? If second reviewer is added, every prod PR needs their approval — slower but safer.
  2. Is conjurehq/monorepo the right name, or should we name it conjurehq/conjure-v2 / conjurehq/platform? Renaming later is cheap but breaks bookmarks.
  3. Migrate open PRs and issues from conjurehq/conjure? GitHub has no native transfer; need a script (gh issue list | xargs gh issue create). Decide whether the cost is worth it for outstanding work.
  4. Repository visibility: stay private (default) or open-source after migration? If open-sourcing, LFS bandwidth on forks becomes a real cost — switch to R2 for large assets first.
